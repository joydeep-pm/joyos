# FY26 Win/Loss Analysis

## Executive reading

The dominant story in the FY26 loss data is that revenue leakage was concentrated, explainable, and at least partially recoverable.

The losses do not read like random market noise. They cluster around:
- a handful of product parity gaps
- enterprise/compliance weaknesses
- implementation and integration confidence issues
- trust/incumbency dynamics

## Known loss base

From the source corpus:
- confirmed FY26 India losses: **₹100.5 Cr**
- 23 deals in the tracked India loss set
- 78.5% demo loss rate across 112 demos

## Most important root-cause buckets

### 1. Core feature / parity gaps
This is the largest visible bucket.

Repeated problem areas:
- Gold Lending
- LAS / LAMF
- BNPL / Credit Line
- Vehicle / dealer / lease / fleet finance
- mortgage / collateral-rich lifecycle depth in selected deals

### 2. Compliance and governance gaps
Some deals were blocked not because Finflux lacked generic capability, but because it lacked enterprise confidence on regulatory/accounting expectations.

Repeated themes:
- ECL / provisioning maturity
- EIR / revenue recognition rigor
- DPDPA / InfoSec readiness
- auditability and governance depth

### 3. Trust / incumbency / reference weakness
The source documents explicitly note that a meaningful share of losses are driven by non-product factors.

Repeated themes:
- incumbent comfort
- weak proof of successful comparable deployments
- lack of confidence in implementation outcomes
- weaker enterprise trust versus category specialists

### 4. Integration and implementation complexity
Some losses show that the platform story failed before feature scope could fully rescue the opportunity.

Repeated themes:
- build-vs-buy decisions
- integration complexity vetoes
- poor config/setup optics
- low confidence in implementation speed

## Largest visible examples from the workbook

- **Vivamoney** — loss linked to build-vs-buy and integration complexity; mapped to platform spine work
- **Neo Finance** — LAS loss driven by parity gaps and stronger competitor advantage
- **Piramal Capital** — Gold loss tied to weak gold parity, vault/auction/ornament capability gaps
- **Hinduja Leyland** — vehicle/TWL gap tied to dealer/floorplan weakness
- **Kissht** — LAP loss tied to collateral logic and sale-of-asset gaps, alongside trust issues

## Competitor pattern

The losses suggest Finflux is not merely losing to one class of competitor. It loses to:
- horizontal incumbents with stronger enterprise confidence
- category specialists with deeper product maturity
- internal-build alternatives when integration confidence is weak

Named competitors recur across the source material, including:
- A3SL
- Pennant
- Nucleus
- Craft Silicon
- Finbox
- internal build alternatives

## What makes these losses actionable

A strong feature of the FY27 source corpus is that many losses are already mapped to specific EPICs and quarter fixes.

That means the leadership question is not only:
- “why did we lose?”

It is also:
- “which losses are most recoverable if the mapped EPICs ship on time?”

## Recovery logic for FY27

### High-confidence recovery logic
If FY27 delivers well in these areas, the business should improve conversion odds materially:
- Gold
- LAS / LAMF
- BNPL / Credit Line
- core compliance gates
- implementation/config credibility

### Lower-confidence but high-upside recovery logic
These require more than product delivery alone:
- Islamic finance / GCC
- capital markets
- broader international / localization-driven categories

## Leadership implication

Win/loss review should become an operating loop, not a one-time analysis.

Recommended standing questions:
- Which losses this quarter were product-caused vs trust-caused?
- Which top losses are directly recoverable by already-funded EPICs?
- Where are we still underestimating implementation/reference weakness?
- Which category still lacks enough depth to sell honestly?
