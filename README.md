## Quick Start

### 1. Clone the Repository
```bash
git clone https://github.com/amanaiproduct/personal-os.git
cd personal-os
```

### 2. Run Setup (2 minutes)
```bash
./setup.sh
```

The setup will:
- Create your workspace directories
- Guide you through questions about your goals and priorities
- Generate your personalized `GOALS.md`
- Copy template files

**Note:** Python 3.10+ is only needed if you want to run the MCP server. The basic setup works with just bash.

### 3. Start Using It
```text
# In your AI assistant (Claude Code, etc.)
"Read AGENTS.md and help me run this as a Director of Products operating system"
```

## How It Works

1. **Capture quickly** — dump raw items into `BACKLOG.md`
2. **Process with context** — the assistant turns notes into structured tasks using `GOALS.md` and `Knowledge/`
3. **Intervene deliberately** — ask what needs your attention and get a Director Intervention Brief plus **Today’s Three**
4. **Compound context** — store feature requests, meetings, people notes, and learnings in `Knowledge/`
5. **Review weekly** — reflect on what moved Documentation, Stability, New Business, and Team Leadership

### Common Commands

- `"Process my backlog"` — turn raw notes into structured tasks
- `"What needs my intervention today?"` — get a director intervention brief
- `"What are my Today's Three?"` — keep daily focus constrained
- `"Prep my 1:1 with [PM name]"` — prepare a people-management conversation
- `"Turn this meeting note into follow-ups"` — extract action items and updates
- `"What moved Documentation, Stability, or New Business this week?"` — drive weekly reflection

### Priorities

| Priority | Meaning | Limit |
|----------|---------|-------|
| **P0** | Do today / director attention required | max 3 |
| **P1** | This week | max 7 |
| **P2** | Scheduled | - |
| **P3** | Someday / maybe | - |

```text
│
├── Tasks/                   # Personal tasks (gitignored)
├── Knowledge/               # Feature requests, meetings, people, learnings (gitignored)
├── BACKLOG.md               # Raw capture inbox (gitignored)
├── GOALS.md                 # Current goals and priorities (generated by setup)
├── AGENTS.md                # Assistant operating instructions
└── setup.sh                 # Interactive setup script
```

## Daily Workflow

**Morning:**
- Ask: `"What needs my intervention today?"`
- Review the **Director Intervention Brief**
- Commit to **Today’s Three** only

**During work:**
- Brain dump into `BACKLOG.md`
- Save durable context to `Knowledge/Feature-Requests/`, `Knowledge/Meetings/`, `Knowledge/People/`, and `Knowledge/Learnings/`
- Convert meeting notes and escalations into tasks and follow-ups

**End of day:**
- Mark completed tasks
- Capture unresolved blockers or fresh escalations into backlog or knowledge notes

**Weekly:**
- Run a weekly review
- Check what moved Documentation, Stability, New Business, and Team Leadership
- Update learnings and stale follow-ups

## Features

- Goal-driven prioritization based on your real operating goals
- Director-focused daily planning with **Today’s Three**
- Smart deduplication for backlog processing
- Markdown-first knowledge model for feature requests, meetings, people, and learnings
- Session evals to review and improve AI interactions over time
- Optional MCP integration for Claude and other AI assistants
