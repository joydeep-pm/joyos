# Learnings

Use this folder to capture recurring patterns the Personal OS should learn from over time.

## What belongs here
- repeated blocker causes
- recurring documentation gaps
- prompting or workflow mistakes
- repeated grooming failures
- process improvements worth adopting
- patterns that keep showing up across PMs, meetings, or escalations

## Suggested structure for each note
- What happened
- Why it mattered
- Pattern observed
- What should change next time
- Related tasks, meetings, or people

## Guidance
- Treat repeated issues as system signals, not isolated anecdotes.
- Keep these notes concise and actionable.
- Review them during the weekly review and use them to improve how the workspace operates.
