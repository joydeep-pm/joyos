# Feature Requests

Use this folder for durable notes about major feature asks, initiatives, client-driven requests, and escalations.

## What belongs here
- feature requests that need multiple follow-ups
- client asks that should not live only in meeting notes
- initiatives that need clearer ownership, blockers, or next decisions
- items that may feed grooming, leadership updates, or roadmap conversations

## Suggested structure for each note
- Summary
- Source / client / stakeholder
- Product charter
- PM owner
- Current stage
- Risks / blockers
- Next intervention needed
- Related links or meeting references

## Guidance
- Prefer one durable note per meaningful request instead of duplicating context across many files.
- Update the same file over time as the request evolves.
- Link relevant tasks back to the note using `resource_refs`.
